# 收货地址 API

> 会员收货地址管理相关接口，支持多地址管理和默认地址切换。

## 1. 概述

| 项目 | 说明 |
|------|------|
| C端 Controller | `app/address/AppAddressController` |
| Service | `AddressService` |
| 路径前缀 | `/member/app/address` |

## 2. C端接口

### 2.1 查询收货地址列表

| 项目 | 说明 |
|------|------|
| 说明 | 获取当前登录用户的所有收货地址 |

### 2.2 查询收货地址详情

| 项目 | 说明 |
|------|------|
| 说明 | 根据地址 ID 获取地址详情 |

### 2.3 创建收货地址

| 项目 | 说明 |
|------|------|
| 说明 | 新增收货地址 |

**字段说明：**

| 字段 | 说明 |
|------|------|
| name | 收件人名称 |
| mobile | 手机号 |
| area_id | 地区 ID（省市区） |
| detail_address | 详细地址 |
| default_status | 是否默认地址 |

**业务逻辑：**

- 如果设置为默认地址，自动取消其他地址的默认状态
- 首个地址自动设为默认地址

### 2.4 修改收货地址

| 项目 | 说明 |
|------|------|
| 说明 | 修改已有地址信息 |

### 2.5 删除收货地址

| 项目 | 说明 |
|------|------|
| 说明 | 删除指定地址 |

**业务逻辑：**

- 如果删除的是默认地址，自动将最近使用的地址设为默认

### 2.6 设置默认地址

| 项目 | 说明 |
|------|------|
| 说明 | 将指定地址设为默认地址 |

**业务逻辑：**

- 取消当前默认地址的默认状态
- 设置新地址为默认

## 3. 跨模块 API

`MemberAddressApi` 为其他模块提供的接口：

| 方法 | 说明 | 消费方 |
|------|------|--------|
| getAddress(Long addressId) | 根据 ID 查询地址详情 | mall-trade |
| getDefaultAddress(Long userId) | 查询用户默认地址 | mall-trade |

## 4. 数据模型

### member_address 表

| 字段 | 类型 | 说明 |
|------|------|------|
| id | Long | 地址 ID |
| user_id | Long | 用户 ID |
| name | String | 收件人名称 |
| mobile | String | 手机号 |
| area_id | Long | 地区 ID（关联地区表） |
| detail_address | String | 详细地址 |
| default_status | Boolean | 是否默认地址 |

## 5. 注意事项

- 一个用户可以拥有多个收货地址
- 同一用户同一时间只能有一个默认地址
- 删除默认地址时需要自动选择新的默认地址
- 地址的 area_id 关联系统的地区表，用于省市区联动选择
- mall-trade 模块下单时通过 `MemberAddressApi.getDefaultAddress` 获取默认地址
