# 标签与分组管理 API

> 会员标签和分组的管理接口，用于会员分类管理。

## 1. 标签管理

### 1.1 概述

| 项目 | 说明 |
|------|------|
| Controller | `admin/tag/MemberTagController` |
| Service | `MemberTagService` |
| 路径前缀 | `/member/tag` |
| 权限前缀 | `member:tag:*` |

### 1.2 接口清单

#### 查询标签列表

| 项目 | 说明 |
|------|------|
| 权限 | `member:tag:query` |
| 说明 | 查询所有会员标签 |

#### 创建标签

| 项目 | 说明 |
|------|------|
| 权限 | `member:tag:create` |
| 说明 | 新增会员标签 |

#### 修改标签

| 项目 | 说明 |
|------|------|
| 权限 | `member:tag:update` |
| 说明 | 修改标签名称 |

#### 删除标签

| 项目 | 说明 |
|------|------|
| 权限 | `member:tag:delete` |
| 说明 | 删除标签 |

**错误码：**

| 错误码 | 说明 |
|--------|------|
| 1_004_006_000 | 用户标签不存在 |

### 1.3 数据模型

**member_tag 表：**

| 字段 | 类型 | 说明 |
|------|------|------|
| id | Long | 标签 ID |
| name | String | 标签名称 |

**标签与用户的关系：**

- 标签与用户是 N:N 关系
- 用户的标签 ID 列表以 JSON 数组形式存储在 `member_user.tag_ids` 字段中
- 查询用户标签时需要根据 tag_ids 反查标签表

### 1.4 标签使用说明

- 标签用于对会员进行分类标记（如：VIP、活跃用户、新用户等）
- 一个用户可以拥有多个标签
- 标签可在营销活动中用于精准人群筛选

## 2. 分组管理

### 2.1 概述

| 项目 | 说明 |
|------|------|
| Controller | `admin/group/MemberGroupController` |
| Service | `MemberGroupService` |
| 路径前缀 | `/member/group` |
| 权限前缀 | `member:group:*` |

### 2.2 接口清单

#### 查询分组列表

| 项目 | 说明 |
|------|------|
| 权限 | `member:group:query` |
| 说明 | 查询所有会员分组 |

#### 创建分组

| 项目 | 说明 |
|------|------|
| 权限 | `member:group:create` |
| 说明 | 新增会员分组 |

#### 修改分组

| 项目 | 说明 |
|------|------|
| 权限 | `member:group:update` |
| 说明 | 修改分组信息 |

#### 删除分组

| 项目 | 说明 |
|------|------|
| 权限 | `member:group:delete` |
| 说明 | 删除分组（需确保无用户关联） |

### 2.3 数据模型

**member_group 表：**

| 字段 | 类型 | 说明 |
|------|------|------|
| id | Long | 分组 ID |
| name | String | 分组名称 |
| remark | String | 备注 |
| status | Integer | 状态（0-启用，1-禁用） |

**分组与用户的关系：**

- 分组与用户是 N:1 关系
- 用户的分组 ID 存储在 `member_user.group_id` 字段中
- 一个用户只能属于一个分组

### 2.4 分组使用说明

- 分组用于对会员进行业务分组（如：普通会员、企业会员等）
- 分组与标签的区别：标签是多对多，分组是多对一
- 分组可影响会员的差异化服务策略

---

## 3. 标签与分组的区别

| 维度 | 标签 (Tag) | 分组 (Group) |
|------|-----------|-------------|
| 关系 | N:N（多对多） | N:1（多对一） |
| 存储 | JSON 数组在 user 表 | 外键在 user 表 |
| 用途 | 灵活分类标记 | 业务分组归属 |
| 数量 | 一个用户可有多个标签 | 一个用户只属于一个分组 |
| 示例 | VIP、活跃用户、高消费 | 普通组、企业组、渠道组 |

## 4. 注意事项

- 删除标签前应检查是否有用户引用该标签
- 删除分组前应检查是否有用户属于该分组
- 标签和分组的增删改不影响已关联用户的数据（标签 JSON 存储，分组外键关联）
- 批量更新用户标签时需要注意并发安全
