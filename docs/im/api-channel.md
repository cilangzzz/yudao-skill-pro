# 频道与素材接口

## 概述

频道是 IM 模块中的单向广播能力，管理员通过频道发布素材内容推送给指定用户群。

### 业务定位

- 频道是单向广播通道，区别于群聊的双向交互
- 素材是频道发布的内容载体，支持图文混排
- 频道消息将素材推送给指定用户列表

### 核心实体

| 实体 | 说明 |
|-----|------|
| ChannelDO | 频道定义，包含编码、名称、头像、排序、状态 |
| ChannelMaterialDO | 频道素材，包含标题、封面、摘要、HTML 内容、链接地址 |
| ChannelMessageDO | 频道广播消息，关联素材并指定接收用户 |

## 频道素材接口

### 1. 获取频道素材

- **路径**: `GET /im/channel/material/get`
- **说明**: 获取指定频道素材详情
- **权限**: 需要用户登录
- **请求参数**:

| 参数 | 类型 | 必填 | 说明 |
|-----|------|-----|------|
| id | Long | 是 | 素材 ID |

- **响应**: `CommonResult<ChannelMaterialDO>` - 素材详情
- **实现逻辑**:
  1. 查询 `im_channel_material` 表获取素材记录
  2. 返回素材的完整信息（标题、封面、摘要、内容 HTML、链接）

## 关键实现

- **ChannelService**: 频道管理服务，管理频道的 CRUD 和状态
- **ChannelMaterialService**: 素材管理服务，管理素材的创建、编辑、查询
- **ChannelMessageService**: 频道消息服务，管理广播消息的发送和拉取
- **素材与消息分离**: 素材是内容载体，消息是推送记录，通过 `materialId` 关联
- **接收者指定**: 频道消息的 `receiverUserIds` 字段指定接收用户，为空则广播给频道所有关注者

## 与群聊的区别

| 特性 | 频道广播 | 群聊 |
|-----|---------|------|
| 方向 | 单向（管理员 -> 用户） | 双向（所有成员可发言） |
| 消息发送 | 仅管理员可发布 | 所有非禁言成员可发送 |
| 互动 | 无回复功能 | 支持回复、@提醒 |
| 已读 | 不追踪已读状态 | 支持已读回执统计 |
| 适用场景 | 公告、资讯推送 | 日常交流讨论 |
