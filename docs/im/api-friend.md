# 好友与好友请求接口

## 概述

好友管理是 IM 模块的基础功能，提供好友关系维护和好友申请审批能力。

### 业务定位

- 双向好友关系模型，每对好友在数据库中存储 2 行记录（A->B 和 B->A）
- 每个方向独立管理 silent（免打扰）、pinned（置顶）、blocked（拉黑）、displayName（备注名）
- 好友请求支持申请、同意、拒绝的完整审批流程

### 核心实体

| 实体 | 说明 |
|-----|------|
| FriendDO | 好友关系记录，包含 userId、friendId 及个性化属性（免打扰/置顶/拉黑/备注名） |
| FriendRequestDO | 好友请求记录，包含申请内容、来源、处理状态 |

## 好友接口列表

### 1. 获取好友列表

- **路径**: `GET /im/friend/list`
- **说明**: 获取当前登录用户的好友列表
- **权限**: 需要用户登录
- **请求参数**:

| 参数 | 类型 | 必填 | 说明 |
|-----|------|-----|------|
| - | - | - | 无额外参数，基于当前登录用户查询 |

- **响应**: `CommonResult<List<FriendDO>>` - 好友列表
- **实现逻辑**:
  1. 获取当前登录用户 ID
  2. 查询 `im_friend` 表中 `userId = 当前用户` 的所有记录
  3. 返回好友列表

### 2. 获取单个好友

- **路径**: `GET /im/friend/get`
- **说明**: 获取指定好友的信息
- **权限**: 需要用户登录
- **请求参数**:

| 参数 | 类型 | 必填 | 说明 |
|-----|------|-----|------|
| friendId | Long | 是 | 好友用户 ID |

- **响应**: `CommonResult<FriendDO>`

### 3. 删除好友

- **路径**: `DELETE /im/friend/delete`
- **说明**: 删除好友关系（双向删除，删除 2 行记录）
- **权限**: 需要用户登录
- **请求参数**:

| 参数 | 类型 | 必填 | 说明 |
|-----|------|-----|------|
| friendId | Long | 是 | 好友用户 ID |

- **响应**: `CommonResult<Boolean>`
- **实现逻辑**:
  1. 校验好友关系存在
  2. 删除 `im_friend` 表中 `userId=A, friendId=B` 和 `userId=B, friendId=A` 两条记录

### 4. 更新好友信息

- **路径**: `PUT /im/friend/update`
- **说明**: 更新好友的个性化信息（备注名、免打扰、置顶等）
- **权限**: 需要用户登录
- **请求参数**:

| 参数 | 类型 | 必填 | 说明 |
|-----|------|-----|------|
| friendId | Long | 是 | 好友用户 ID |
| displayName | String | 否 | 好友备注名 |
| silent | Boolean | 否 | 是否免打扰 |
| pinned | Boolean | 否 | 是否置顶 |

- **响应**: `CommonResult<Boolean>`
- **注意**: 仅更新当前用户方向的记录（A->B），不影响对方的记录（B->A）

### 5. 拉黑好友

- **路径**: `PUT /im/friend/block`
- **说明**: 拉黑指定好友
- **权限**: 需要用户登录
- **请求参数**:

| 参数 | 类型 | 必填 | 说明 |
|-----|------|-----|------|
| friendId | Long | 是 | 好友用户 ID |

- **响应**: `CommonResult<Boolean>`
- **实现逻辑**:
  1. 校验好友关系存在
  2. 设置当前用户方向记录的 `blocked = true`

### 6. 取消拉黑

- **路径**: `PUT /im/friend/unblock`
- **说明**: 取消拉黑指定好友
- **权限**: 需要用户登录
- **请求参数**:

| 参数 | 类型 | 必填 | 说明 |
|-----|------|-----|------|
| friendId | Long | 是 | 好友用户 ID |

- **响应**: `CommonResult<Boolean>`

## 好友请求接口列表

### 7. 申请添加好友

- **路径**: `POST /im/friend-request/apply`
- **说明**: 向目标用户发送好友申请
- **权限**: 需要用户登录
- **请求参数**:

| 参数 | 类型 | 必填 | 说明 |
|-----|------|-----|------|
| friendId | Long | 是 | 目标用户 ID |
| applyContent | String | 否 | 申请附言 |
| displayName | String | 否 | 对好友的备注名 |
| addSource | Integer | 否 | 添加来源（搜索/扫码/群聊等） |

- **响应**: `CommonResult<Long>` - 好友请求 ID
- **实现逻辑**:
  1. 校验目标用户存在
  2. 校验是否已经是好友
  3. 校验是否已存在未处理的请求
  4. 创建好友请求记录

### 8. 同意好友请求

- **路径**: `PUT /im/friend-request/agree`
- **说明**: 同意好友申请，建立双向好友关系
- **权限**: 需要用户登录
- **请求参数**:

| 参数 | 类型 | 必填 | 说明 |
|-----|------|-----|------|
| id | Long | 是 | 好友请求 ID |

- **响应**: `CommonResult<Boolean>`
- **实现逻辑**:
  1. 校验请求存在且未处理
  2. 更新请求状态为"已同意"
  3. 创建双向好友关系（插入 2 行 `im_friend` 记录）
  4. 事务内完成，保证数据一致性

### 9. 拒绝好友请求

- **路径**: `PUT /im/friend-request/refuse`
- **说明**: 拒绝好友申请
- **权限**: 需要用户登录
- **请求参数**:

| 参数 | 类型 | 必填 | 说明 |
|-----|------|-----|------|
| id | Long | 是 | 好友请求 ID |

- **响应**: `CommonResult<Boolean>`

### 10. 获取好友请求列表

- **路径**: `GET /im/friend-request/list`
- **说明**: 获取当前用户收到的好友申请列表
- **权限**: 需要用户登录
- **请求参数**:

| 参数 | 类型 | 必填 | 说明 |
|-----|------|-----|------|
| pageNo | Integer | 是 | 页码 |
| pageSize | Integer | 是 | 每页大小 |

- **响应**: `CommonResult<PageResult<FriendRequestDO>>`

### 11. 获取单个好友请求

- **路径**: `GET /im/friend-request/get`
- **说明**: 获取指定好友请求详情
- **权限**: 需要用户登录
- **请求参数**:

| 参数 | 类型 | 必填 | 说明 |
|-----|------|-----|------|
| id | Long | 是 | 好友请求 ID |

- **响应**: `CommonResult<FriendRequestDO>`

## 关键实现

- **FriendService**: 好友关系服务，管理好友的增删改查、拉黑/取消拉黑
- **FriendRequestService**: 好友请求服务，管理申请、同意、拒绝流程
- **双向删除**: 删除好友时同时删除两个方向的记录，保证数据一致性
- **事务保护**: 同意好友请求时，更新请求状态和创建好友关系在同一事务中完成
