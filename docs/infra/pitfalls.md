# 注意事项

## 文件存储相关

### 1. FileClient 缓存刷新延迟

- FileClient 使用 Guava LoadingCache 缓存，10 秒异步刷新
- 修改文件配置后，最多 10 秒内生效
- 不要期望配置修改立即生效，避免在同一请求中修改配置后立即使用

### 2. 文件路径生成规则

- 使用 `generateUploadPath()` 方法生成唯一路径
- 路径包含日期前缀和时间戳后缀，避免文件名冲突
- 不要自行拼接文件路径，避免路径冲突和安全问题

### 3. 主配置唯一性

- `infra_file_config` 表的 `master` 字段标识是否为主配置
- 系统中应只有一个主配置，用于默认文件上传
- 创建新的主配置时，旧的主配置会被自动取消

### 4. 数据库存储方式的限制

- `infra_file_content` 表使用 LONGBLOB 存储文件内容
- 单个文件最大支持约 4GB（MySQL LONGBLOB 限制）
- 大文件不建议使用数据库存储方式，应使用 S3/本地存储

### 5. FileClientFactory 初始化失败

- FileClient 初始化时会连接存储服务
- 如果存储服务不可用，初始化会失败，对应 FileClient 不会被缓存
- 需要确保存储服务可用后再创建/修改文件配置

## 代码生成相关

### 6. 表名命名规范

- 代码生成器从数据库表名推导 Java 类名和业务名
- 表名应遵循 `模块_业务名` 格式，如 `system_user`、`infra_file`
- 不规范的表名会导致生成的类名和包名不符合预期

### 7. 主键字段要求

- 代码生成器默认使用名为 `id` 的字段作为主键
- 主键字段必须设置为 `primary_key = true`
- 非标准主键字段需要在列配置中手动标记

### 8. 子表关联字段

- 主子表场景下，子表必须有外键关联到主表
- `sub_join_column_id` 指向子表中的关联字段
- `sub_join_many` 标识关联关系是一对一还是一对多

### 9. 数据源连接安全

- `infra_data_source_config` 表中的密码字段使用加密存储
- 不要在日志或响应中暴露数据源密码
- 测试数据源连接时，验证结果不包含密码信息

## 定时任务相关

### 10. JobHandler 幂等性

- `JobHandler.execute()` 方法应实现幂等性
- 任务可能因为重试机制被多次执行
- 不要依赖"只执行一次"的假设，避免重复执行产生副作用

### 11. CRON 表达式验证

- 创建或更新定时任务时，CRON 表达式会被验证格式
- 无效的 CRON 表达式会导致任务创建失败
- 使用在线 CRON 表达式工具验证后再配置

### 12. 任务状态管理

- 任务状态变更会同步到 Quartz Scheduler
- 暂停的任务不会自动恢复，需要手动恢复
- 删除任务会同时删除 Quartz 中的任务和触发器

### 13. 任务执行超时

- `monitor_timeout` 字段配置监控超时时间
- 超时后任务状态会被标记为异常
- 合理设置超时时间，避免误报

## API 日志相关

### 14. 日志数据量控制

- API 日志表数据量会持续增长
- 需要定期清理历史日志数据
- 建议配置日志保留策略，自动清理过期数据

### 15. 敏感信息脱敏

- `request_params` 和 `response_body` 字段可能包含敏感信息
- 日志记录时应进行脱敏处理（如密码、token 等）
- 查看日志时注意信息安全

### 16. traceId 关联

- `trace_id` 字段用于关联访问日志和错误日志
- 确保请求链路中 traceId 正确传递
- 排查问题时，通过 traceId 可以找到完整的请求链路

## 系统配置相关

### 17. 配置缓存

- ConfigService 可能对配置值进行缓存
- 修改配置后，缓存刷新有一定延迟
- 不要在同一请求中修改配置后立即读取新值

### 18. 配置键名规范

- 配置键名应遵循统一命名规范
- 推荐使用 `模块.功能.参数` 格式，如 `sys.user.init-password`
- 避免使用特殊字符和过长的键名

## 通用注意事项

### 19. 租户隔离

- infra 模块大部分表使用 `@TenantIgnore` 注解，不参与租户隔离
- 文件配置、代码生成、定时任务等数据在所有租户间共享
- 不要在这些表中存储租户相关的业务数据

### 20. 错误码规范

- infra 模块错误码前缀为 `1-001-xxx-xxx`
- 常见错误码：
  - `1_001_000_001` - 参数配置不存在
  - `1_001_001_000` - 定时任务不存在
  - `1_001_003_001` - 文件不存在
  - `1_001_004_002` - 表定义已经存在
  - `1_001_006_000` - 文件配置不存在

### 21. 性能优化

- 文件查询使用 `likeIfPresent` 进行模糊匹配，大数据量下注意性能
- 日志查询建议按时间范围过滤，避免全表扫描
- 代码生成表的列定义查询建议使用 `table_id` 索引
