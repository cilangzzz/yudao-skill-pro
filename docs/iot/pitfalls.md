# IoT 模块注意事项与常见问题

> 本文档汇总 IoT 模块开发和运维中的关键注意事项、常见陷阱和最佳实践。

## 设备密钥安全

- 设备密钥（deviceSecret）必须使用强随机数生成，禁止硬编码或使用简单规则生成
- 设备密钥在传输和存储过程中应加密保护
- 产品密钥（productSecret）用于动态注册签名验证，严禁泄露给终端设备
- 设备认证失败时不应返回具体失败原因，避免信息泄露

## 协议扩展注意事项

- 新协议必须实现 `IotProtocol` 接口的所有方法：`getId()`、`getServerId()`、`getType()`、`start()`、`stop()`、`isRunning()`
- 下行消息订阅者应继承 `AbstractIotProtocolDownstreamSubscriber`，复用通用逻辑
- 协议类型枚举需在 `IotProtocolTypeEnum` 中注册，否则无法被协议管理器识别
- 协议实现必须处理 `start()` 和 `stop()` 的资源释放，避免端口泄漏

## 消息存储与查询

- 设备消息使用 TDengine 时序数据库存储，不支持标准 SQL 的所有语法
- 消息查询应注意时间范围限制，避免全表扫描
- 大量消息写入时应使用批量接口，减少数据库压力
- TDengine 与 MySQL 的 SQL 语法有差异，开发时需注意兼容性

## 设备缓存策略

- 频繁访问的设备信息（认证、状态查询）使用 Redis 缓存
- 缓存键格式：`productKey + deviceName` 组合查询优先于 `deviceId` 查询
- 设备状态变更时必须同步更新缓存，否则会导致状态不一致
- 缓存失效策略应合理设置，避免缓存穿透

## 规则引擎性能

- 场景规则应设置合理的触发条件，避免频繁触发导致系统过量负载
- 规则匹配使用责任链模式，匹配器顺序会影响性能，高频判断条件应前置
- 大量规则同时启用时注意监控规则引擎的执行耗时
- 规则中的动作执行应考虑异步化，避免阻塞规则引擎线程

## OTA 升级风险

- 大规模升级必须分批进行，避免同时下发导致网络拥塞和服务器压力
- 固件文件必须校验签名（file_digest_value），防止篡改
- 升级任务应支持暂停和取消，应对异常情况
- 升级记录用于追溯问题，不应随意删除

## 租户隔离

- 所有设备、产品、场景规则等操作都必须考虑租户隔离
- 继承 `TenantBaseDO` 的实体自动支持租户过滤，继承 `BaseDO` 的实体需手动处理
- 设备认证时需校验设备所属租户，防止越权访问
- 跨租户数据查询是严重的安全漏洞，开发时务必注意

## 设备动态注册

- 产品需先开启 `register_enabled` 才允许动态注册
- 动态注册的签名算法基于 `productSecret`，需确保签名验证逻辑正确
- 动态注册创建的设备默认状态为"未激活"，需后续认证激活
- 限制同一产品的动态注册频率，防止恶意注册

## 消息总线使用

- `IotMessageBus` 是 gateway 和 biz 层之间的唯一通信通道
- 上行消息 topic：`device:message:upstream`
- 下行消息 topic：`device:message:downstream`
- 消息体应包含完整的设备标识（productKey + deviceName）以便路由

## 常见错误码

| 错误码 | 说明 | 处理建议 |
|-------|------|---------|
| 1-011-000-000 | 设备不存在 | 检查设备 ID 或 productKey + deviceName 是否正确 |
| 1-011-000-001 | 设备认证失败 | 检查 deviceSecret 是否正确，设备是否已激活 |
| 1-011-000-002 | 设备已存在 | 设备名称在产品内已重复，更换设备名称 |
| 1-011-001-000 | 产品不存在 | 检查产品 ID 或 productKey 是否正确 |
| 1-011-002-000 | 物模型不存在 | 检查物模型功能 ID 或标识符是否正确 |

## 外部依赖注意事项

| 依赖 | 注意事项 |
|-----|---------|
| vertx-mqtt | MQTT 服务器基于 Vert.x 异步框架，回调中避免阻塞操作 |
| californium | CoAP 服务器使用 Eclipse Californium，注意 DTLS 配置 |
| jmodbus | Modbus 协议支持 TCP 客户端和服务端两种模式 |
| netty | TCP/UDP/WebSocket 底层框架，注意 ByteBuf 的释放避免内存泄漏 |
| tdengine-jdbc | TDengine 驱动版本需与服务端匹配，3.x 驱动不兼容 2.x 服务端 |
