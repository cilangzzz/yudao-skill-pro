# 注意事项

## 库存并发安全

### 1. 库存更新必须使用行锁

- 库存更新必须使用 `SELECT ... FOR UPDATE` 行锁锁定库存记录
- 不使用行锁会导致并发更新丢失，例如两个出库单同时扣减同一 SKU 库存
- 示例：`SELECT * FROM wms_inventory WHERE sku_id = ? AND warehouse_id = ? FOR UPDATE`

### 2. 库存懒初始化的并发处理

- 首次入库时，目标 SKU + 仓库的库存记录可能不存在
- 使用 `INSERT ... ON DUPLICATE KEY UPDATE` 或捕获 `DuplicateKeyException` 实现并发安全的懒初始化
- 流程：先尝试 INSERT，若捕获 DuplicateKeyException 则重新 SELECT FOR UPDATE 后 UPDATE
- 唯一索引 `uk_sku_warehouse(sku_id, warehouse_id)` 是此机制的前提

### 3. 移库双向更新的死锁风险

- 移库操作同时更新源仓库和目标仓库的库存
- 如果多个移库单涉及相同的 SKU + 仓库组合，可能产生死锁
- 建议：按仓库 ID 升序获取锁，确保锁获取顺序一致

### 4. 出库库存校验

- 完成出库前必须校验库存数量足够
- 校验和扣减必须在同一个事务中，使用行锁保护
- 不要先查询再更新（存在 TOCTOU 竞态条件）

## 单据生命周期

### 5. 单据状态机

- 所有单据遵循统一状态机：PREPARE(0) -> FINISHED(4) 或 CANCELED(5)
- 只有 PREPARE 状态的单据才能执行完成或取消操作
- FINISHED 和 CANCELED 状态为终态，不可再变更
- 操作前必须校验单据状态，避免重复完成或取消

### 6. 单据完成的事务一致性

- 单据完成操作涉及多个步骤：状态更新 + 库存变更 + 历史记录
- 所有步骤必须在同一个事务中完成
- 任何一步失败都应回滚整个事务

### 7. 单据删除的限制

- 只有 PREPARE 状态的单据才能删除
- 删除单据时必须同时删除关联的明细行
- FINISHED 状态的单据不应被删除，因为已经产生了库存变动

## SKU 管理

### 8. SKU 作为库存单位

- 库存以 SKU + 仓库维度跟踪，不跟踪物料级别库存
- 物料创建时必须包含至少一个 SKU
- 删除物料会级联删除所有关联 SKU，需确保无库存记录

### 9. SKU 嵌入管理

- SKU 通过物料接口管理（创建/更新物料时一起处理）
- 更新物料时采用先删后插的方式同步 SKU
- 独立的 SKU 查询接口为只读，不支持直接 CRUD

### 10. SKU 编码唯一性

- SKU 编号（sku_no）应保证唯一性
- 建议在数据库层面添加唯一索引约束

## 订单类型常量

### 11. WmsOrderTypeConstants 分段设计

- 入库单类型范围：[100, 200)
- 出库单类型范围：[200, 300)
- 移库单类型范围：[300, 400)
- 盘点单类型范围：[400, 500)
- 新增单据类型时必须在对应范围内分配，避免类型冲突

### 12. 库存历史来源标识

- 库存变动历史的 sourceOrderType 和 sourceOrderId 用于追溯库存变动来源
- 完成单据时必须正确填写这两个字段
- 查询库存历史时可通过来源单据类型和编号过滤

## VO 组装模式

### 13. 批量查询避免 N+1

- VO 组装时必须使用批量查询（如 `getUserMap()`），避免循环中逐条查询
- 典型模式：先查询列表 -> 批量获取关联实体 Map -> 遍历列表组装 VO
- 使用 `BeanUtils.toBean()` 进行对象转换

### 14. AdminUserApi 跨模块调用

- 通过 `AdminUserApi.getUserMap()` 批量获取用户信息
- 注意处理用户不存在的情况（Map 中无对应 key）
- 用户信息用于 VO 中展示创建人、更新人名称

## Excel 导出

### 15. 导出数据量控制

- Excel 导出应注意数据量控制，避免内存溢出
- 大数据量导出建议使用 EasyExcel 的流式写入
- 导出查询建议限制最大记录数或分批导出

### 16. 导出字段映射

- 导出字段使用 `@ExcelProperty` 注解映射
- 关联字段（如仓库名称、商户名称）需要在导出前组装
- 枚举字段需要转换为中文描述

## 错误码规范

### 17. WMS 模块错误码

- WMS 模块错误码需遵循项目统一的错误码分配规则
- 常见错误码：
  - 物料不存在
  - 仓库不存在
  - 商户不存在
  - SKU 不存在
  - 库存不足
  - 单据状态不允许操作
  - 品牌/分类存在关联数据不允许删除

## 性能优化

### 18. 库存查询索引

- 库存表使用 `uk_sku_warehouse(sku_id, warehouse_id)` 唯一索引
- 库存历史表使用 `idx_sku_warehouse(sku_id, warehouse_id)` 联合索引
- 按时间范围查询库存历史时，建议配合 createTime 索引

### 19. 单据查询优化

- 单据表按 no、warehouse_id、status 建立索引
- 分页查询建议按状态过滤，缩小扫描范围
- 明细表按 order_id 建立索引，通过 order_id 批量查询明细

### 20. 分类树查询

- 分类表使用 parentId 实现树形结构
- 树形查询建议一次性加载所有分类到内存后构建树
- 避免递归查询数据库
